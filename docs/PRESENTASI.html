1. Latar Belakang

Serangan Man-in-the-Middle (MITM) mengeksploitasi kelemahan protokol ARP yang stateless dan tanpa autentikasi


🔴 Ancaman

  • ARP tidak memverifikasi pengirim
  • HTTP mengirim data plaintext
  • Credential theft via sniffing

🟢 Solusi

  • HTTPS / TLS encryption
  • HSTS, JWT, bcrypt
  • Defense in Depth

2. Address Resolution Protocol (ARP)

RFC 826 — Layer 2 (Data Link)

Cara Kerja Normal

  1. Host broadcast ARP Request
  2. Target reply dengan MAC
  3. Disimpan di ARP Cache

⚠️ Kelemahan

  • Stateless — tanpa state tracking
  • No Auth — Reply diterima tanpa verifikasi
  • Gratuitous ARP — Reply tanpa Request

💡 Attacker mengirim ARP Reply palsu → victim percaya MAC attacker = MAC gateway → semua traffic melewati attacker

3 Metode Serangan

A. Scapy + Sniffer

Python script custom

docker compose exec attacker bash

# Terminal 1
python3 arp_spoof.py \
  -t 172.20.0.10 -g 172.20.0.20

# Terminal 2
python3 sniff_credentials.py \
  --mode http --port 5000

B. Bettercap

All-in-one MITM

docker compose exec attacker bash

bettercap \
  -iface eth0 \
  -caplet mitm_attack.cap

# Auto: ARP Spoof
#     + HTTP Proxy
#     + Credential sniff

C. tshark

CLI packet capture

docker compose exec attacker bash

# Terminal 1: ARP Spoof
python3 arp_spoof.py \
  -t 172.20.0.10 -g 172.20.0.20

# Terminal 2: Capture
tshark -i eth0 \
  -w /tmp/capture.pcap

6. Solusi Keamanan

🔒 Transport

  • TLS/HTTPS (OpenSSL)
  • HSTS (max-age=1yr)
  • Strong Ciphers (AES-256)

🛡️ Application

  • bcrypt (cost=12)
  • JWT (1h expiry)
  • Helmet (CSP, X-Frame)
  • Rate Limiting, Audit Log

🗄️ Data

  • MongoDB (shared instance)
  • vulnerable_db: plaintext
  • secure_db: bcrypt hash
  • Schema validation, TTL

🌐 Network

  • Static ARP / DAI
  • VPN (Tailscale)
  • VLAN Segmentation

Kesimpulan

  1. ARP Spoofing tetap ancaman nyata — protokol ARP stateless tanpa autentikasi [RFC 826]
  2. HTTP sangat rentan — credential terbaca plaintext saat MITM (OWASP A04:2025)
  3. HTTPS/TLS efektif — meskipun ARP Spoofing berhasil, data tetap terenkripsi
  4. Defense in Depth (4 Layer) — Network + Transport + Application + Data
  5. Keamanan = layer aplikasi — DB sama (MongoDB), tapi password plaintext vs bcrypt hash
  6. Container networking perlu perhatian — Docker bridge default rentan ARP spoofing
